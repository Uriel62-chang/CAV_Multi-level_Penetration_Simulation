# 开发路线图 (Development Roadmap)

> 当前版本：**v0.3.0**（阶段 3 已完成）

- [x] **1. 优化脚本可读性，并解决 subprocess 造成的冗余开销** `[v0.1.0]`
  - 脚本间改为函数 import 调用，消除 subprocess 通信开销
  - 统一定义 CLI 参数，增强代码可维护性

- [x] **2. 增加 CAV 可选的高性能跟驰模型** `[v0.2.0]`
  - HV 始终使用 Krauss，CAV 可选：IDM、ACC、CACC
  - 三者均为 SUMO 内置模型，无需手动实现
  - 新增 CLI 参数 `model`（单值），每次仿真为 CAV 指定一种跟驰模型
  - CACC 需额外定义通信参数（见阶段 6）

- [x] **3. 复杂化路网** `[v0.3.0]`
  - 多边形闭环路网生成 + `--net` 参数切换 + LC2013 换道模型
  - [x] **scenario_0**：方形闭环（baseline，回归测试）
  - [x] **scenario_1**：32边形单车道平滑闭环
  - [x] **scenario_2**：32边形双车道 + LC2013 换道模型 + 多车道检测器聚合
  - [x] **scenario_3**：合流 / 车道收缩瓶颈（125m 双→单→双）
  - **未来拓展**：无信号/有信号交叉口（仅保留规划）

- [ ] **4. 设置多维评估模型** `[v0.4.0]`
  - 效率：平均行程时间、通行能力、延误
  - 安全：TTC（碰撞时间）、紧急制动频次、速度方差
    - 紧急制动频次：捕获 SUMO 仿真 stderr 中的 `emergency braking` warning 并计数，写入 CSV 新增列 `emergency_braking_count`
  - 环保：SUMO 内置 HBEFA 排放模型、燃油消耗
  - 量化结果汇总展示于 README

- [ ] **5. 引入 TraCI 接口，动态调控仿真（进阶）** `[v0.5.0]`
  - 作为独立模块，支持开关切换
  - 典型场景：动态车速引导、匝道汇入控制、可变限速
  - 与现有批量仿真框架解耦

- [ ] **6. 针对 CACC 跟驰模型，对其通信能力调参降级以模拟环境干扰（进阶）** `[v0.6.0]`
  - 可配置参数：丢包率、通信延迟、通信范围衰减
  - 降级后的车队稳定性对比分析
